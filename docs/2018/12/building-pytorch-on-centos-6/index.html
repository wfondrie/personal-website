Building PyTorch on CentOS 6

PyTorch is a powerful deep learning framework for Python. However, PyTorch has been built against newer versions of glibc (2.14 and greater) for since its version 0.4.0 release. This causes a problem for those of us stuck on older systems. In my case, we have machines running CentOS 6.8 with glibc 2.12. In this post, I’ll detail how I’ve successfully built PyTorch 0.4.1 from source against glibc 2.12. This may also work for other versions, but your mileage may vary.

Prerequisites

Python. Anaconda Python is likely the best way to go here. I used Anaconda Python 3.7 for this install. Verify that other Python versions are not in your LD_LIBRARY_PATH… this caused me hours of headaches. My system at work loads Python 2.7 and adds it to the LD_LIBRARY_PATH by default

Compiler. In my case, I used gcc 4.9.2 as my compiler. While newer versions can likely be used successfully, gcc 4.9.2 was used in past PyTorch builds and doesn’t appear to cause issues with CUDA.1

CUDA. If you intend to use GPU support, make sure you have a compatible version of CUDA installed and adjust the install accordingly. I used CUDA 9.2.

git. You’ll need some version of git installed to clone the PyTorch source code from GitHub.

Setting up your environment

Activate a conda environment. I chose to make a new conda environment for my PyTorch build, so as to verify all of its prerequisites were met and there were no interferences. If you’ve installed Anaconda Python, its as easy as:

conda create -n pytorch-dev
conda activate pytorch-dev

Install dependency Python packages. On the PyTorch website, they list several dependencies. However, I also found the dependencies for the caffe2 package to be needed. To install all of the prerquisites, I ran:

conda install numpy pyyaml mkl mkl-include setuptools cmake cffi typing future protobuf hypothesis
conda install -c mingfeima mkldnn
conda install -c pytorch magma-cuda92 # Modify for your CUDA version

Set environment variables. This is as per the directions on the PyTorch website. I found that for the build to detect cuDNN, I had to explicitly provide the path.

export CMAKE_PREFIX_PATH="$(dirname $(which conda))/../"
export CUDNN_LIB_DIR="/path/to/cudnn" # Change as needed or omit.

Clone the repository

We’re almost ready to build, but first we need to get the source code. Just clone the repository, checkout the 0.4.1 release, and update the submodules.

git clone --recursive https://github.com/pytorch/pytorch
cd pytorch

# Get version 0.4.1
git checkout v0.4.1
git submodule update --init --recursive

Build PyTorch

Finally, we can build PyTorch from source. Although the original the instructions direct us to install PyTorch directly, I chose to make a pip wheel to install later:

python setup.py bdist_wheel

Note that if a previous build has gone awry, you can clean the repository with python setup.py clean so that you’re ready for a new build.

Install and verify it works

Install. If you are in the pytorch directory, all you need to do to install is run:

# Your file name may be slightly different
pip install dist/torch-0.5.0a0+a24163-cp37-cp37m-linux_x86_64.whl

Verify. To verify the build was successful, move out of the pytorch directory, fire up the Python interpreter, and try out some code.

from __future__ import print_function
import torch
x = torch.rand(5, 3)
print(x)

And that’s it. Hopefully now you’re off to the races with PyTorch.
